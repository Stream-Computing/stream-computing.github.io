希姆算力云平台解决方案

一. 需求背景​

建设私有云平台是构建智算中心的基石，它是对现有IT基础设施的升级，更是面向未来智能化转型的战略选择，将为组织带来运营效率的飞跃、安全保障的强化以及创新能力的激发，为实现可持续发展奠定坚实基础。

二. 平台价值​

提升资源利用率​

私有云平台通过虚拟化技术，实现计算、存储、网络等IT资源的池化管理，使得资源分配更加灵活高效。这不仅能够根据实际需求动态调整资源规模，避免资源闲置，还能够加快新业务的部署速度，缩短产品上市时间，提升组织的市场响应能力。

加强数据安全​

在处理大规模数据计算与智能分析的过程中，数据安全与隐私保护至关重要。私有云环境允许组织在内部网络中控制和管理数据，相较于公有云，可以实施更严格的安全策略和合规措施，确保敏感信息不外泄，满足行业监管要求，增强用户信任。

优化成本效益​

虽然私有云初期投入可能较高，但长期来看，通过资源的高效利用、能源的有效管理以及避免了公有云服务的持续付费模式，能够显著降低总体拥有成本。同时，私有云的可扩展性意味着随着业务增长，无需频繁进行大规模硬件投资，进一步节省成本。私有云平台提供标准化、自动化的开发测试环境，加速软件迭代周期。开发者可以快速获取所需资源，实验新技术，推动业务创新。这种灵活性对于智算中心探索人工智能、大数据分析等前沿领域尤为关键。

四. 功能特性​

资源管理​

资源调度​

支持以下资源调度相关的特性：

  • 统一调度计算、网络、存储类的资源，例如CPU、内存、AI加速卡、显存、磁盘、IP地址等资源。
  • 独享算力节点，支持管理节点标签、节点污点等，确保工作负载被分配到合适的节点。
  • 在集群、租户等级别完成分配资源、设置配额等操作。
  • 以部署、有状态副本集、守护进程集等形式启动工作负载，满足不同业务的请求。
  • 以任务/定时任务的形式创建容器组并申请相关资源，提升调度效率。
  • 以持久卷声明的形式申请存储资源。
  • 以配置字典的形式为应用负载设置环境变量、设置命令参数、创建配置文件等。

负载均衡​

支持以下负载均衡相关的特性：

  • 通过集群网关向外部暴露集群内部的服务。
  • 通过NodePort或者LoadBalance的方式访问应用路由。

自动化部署及排障​

自动化部署​

支持以下自动化部署相关的特性：

  • 支持自动扩缩容，根据负载高低自动增加或减少副本，以满足应用的要求。
  • 满足大模型部署上线的需求，通过配置管理等功能简化应用的部署和升级操作。
  • 配合服务编排发布等功能简化服务的部署和升级操作。

自动化排障​

支持以下自动化排障相关的特性：

  • 健康检查：定期检查容器组、节点等的状态，及时发现故障。
  • 容器组排障：检测到故障时，自动重启或者调度容器组，确保应用的可用性。
  • 节点排障：节点出故障时，节点会被标记为不可调度，确保新创建的工作负载不会调度到有问题的节点。

监控告警​

集群/应用监控​

支持以下监控相关的特性：

  • 提供全面的监控指标，支持从集群和应用等维度了解资源使用情况，发掘优化点。例如：
    • 接口服务监控：展示请求延迟，包括APPLY、DELETE、GET、PATCH、POST、PUT操作请求延迟以及总均值；每秒请求次数的趋势。
    • 物理资源监控：展示CPU用量、内存用量、CPU平均负载、磁盘用量、Inode用量、磁盘吞吐、IOPS、网络流量、容器组状态的趋势。
    • 调度器监控：展示调度次数、调度延迟、调度速率的趋势。
    • AI加速卡资源总览：展示各个算力卡的实时显存总量、利用率以及利用率趋势、温度趋势。
  • 提供可视化界面查看监控指标，降低操作复杂度，支持多维度排序。
  • 兼容主流的第三方监控系统，例如Prometheus等。

告警通知​

支持以下告警相关的特性：

  • 基于多租户、多维度的监控指标告警策略，发送节点、网络、应用等相关资源的告警。
  • 支持按需设置告警规则。
  • 支持在控制台查看告警信息。

租户管理​

租户设置​

支持以下租户相关的设置：

  • 在租户下完成用户创建、角色分配等操作。
  • 在租户下以项目为单位管理资源，确保资源的合理有效使用。

租户资源​

为租户使用资源提供以下支持：

  • 从集群、租户、项目三个级别授权，确保共享资源的同时完全隔离多个级别的不同角色，确保资源安全。
  • 支持为租户设置资源配额，查看租户资源使用情况。
  • 为租户提供私有镜像仓库，租户管理员可以获取云平台镜像或者自行上传镜像。

访问权限控制​

集群访问权限​

  • 支持从集群级别管理集群角色和集群成员。

租户访问权限​

支持以下租户访问相关权限设置：

  • 支持从租户级别管理平台角色和用户。
  • 基于平台角色进行的细粒度身份验证和授权，确保权限最小化，实现对资源的细粒度访问控制，只有授权用户才能访问特定的资源或服务。
  • 支持从多个维度管理权限，例如租户设置、部门设置、项目设置等。

提供合规性支持和审计工具​

合规支持​

支持以下特性以满足合规性的要求：

  • 采用加密、数据备份、网络安全防护等技术手段，确保数据的安全性和隐私保护。
  • 保护用户个人信息，防止发生泄漏、篡改、丢失等情况。
  • 支持以服务账户的形式，为容器组中运行的进程提供了标识，用于在访问API Server时认证身份。

审计日志​

定期审计提早发现和及时纠正潜在问题，并支持以下特性以提供审计所需数据：

  • 以审计日志形式记录集群和应用日志，方便定期审计和追溯，例如用户登录、操作和资源使用情况。
  • 多租户日志管理，不同租户只能看到自己的日志信息，支持日志检索。
  • 支持项目、工作负载、容器等各个级别的操作日志查询。
  • 支持对接日志收集平台，例如Elasticsearch、Kafka、Fluentd等。

容器镜像仓库管理​

镜像仓库​

提供镜像仓库功能并支持以下特性方便管理操作：

  • 支持面向全平台用户的公共镜像仓库以及面向特定租户的私有镜像仓库。
  • 支持自定义镜像管理，用户可以同步自己的镜像包到对应仓库中。
  • 支持镜像版本控制，确保应用程序的稳定性和一致性。

镜像安全​

为确保镜像安全，支持以下特性：

  • 确保镜像可用，支持经过充分验证的优质镜像源。
  • 提供鉴权，避免外部攻击和内部恶意行为的风险，确保镜像在存储和传输过程中的机密性和完整性。
  • 私有仓库必须配合保密字典，保密字典作为镜像仓库凭证控制使用权限。

算力硬件多样化​

硬件调度​

支持以下特性实现屏蔽底层硬件的差异，为上层用户提供一致的使用体验：

  • 通过插件机制发现和使用算力硬件，保证主体稳定性的同时兼顾可扩展性。
  • 将CPU和AI计算卡等资源虚拟化为资源池统一调度，提高调度效率，提供合理的资源配置。

CPU支持​

CPU硬件的支持情况：

  • 支持x86架构的CPU，例如Intel、AMD。
  • 支持ARM架构的CPU，例如飞腾、鲲鹏。

AI计算卡支持​

AI计算卡的支持情况：适配Li-SEE MS120、STCP920等。

五. 项目实施​

关键技术应用​

  • 希姆算力云平台以K8s生态为基础，广泛适配AI加速卡，融合多租户隔离、多任务执行服务，自主研发相关的任务调度。
  • 作为AI基础算力的平台，可以为上层的AI模型、算法、应用服务提供稳定便捷的算力服务、管理以及应用功能。
  • 以Prometheus生态为基础开发适配硬件的监控体系，对推理卡以及云平台各项运行环境、资源、服务等进行实时监控，配合告警策略可以高效地完成运维工作，并提高云平台的稳定性和可靠性。

应用系统运行所需硬件配置建议​

机房建议配置：配备UPS等保障类设备。

应用集群配置：应用集群中的节点分为控制节点（master）与工作节点（work）。

  • 控制节点需要安装平台相关的软件以及组件，以及K8s集群相关的控制组件，例如日志收集、审计日志、监控指标数据、用户数据、ETCD数据等。
  • 根据ETCD一致性算法原理，需要奇数个控制节点。控制节点至少3个，以奇数为准。控制节点和工作节点的数量建议比例为1:20，即一个控制节点对应20个工作节点。

控制节点的最低配置：

  • CPU：32核
  • 内存：128G
  • 存储：400GB SSD
  • 网络带宽：1G

工作节点的最低配置:

  • CPU：16核
  • 内存：64G
  • 存储：400GB SSD
  • 网络带宽：10G

    数据保护：定期备份数据，防止数据丢失。

应用软件配置的详细说明和配置详细清单​

算力平台搭建完成之后需要对其进行相应的数据初始化和检查，包括如下几个方面：

  • 包括管理员角色、账号以及权限数据初始化。
  • 测试租户数据初始化以及功能验证等。
  • 工作节点相关的功能插件以及监控插件健康检查以及配置等。
  • 操作审计日志辅助软件初始化以及配置等。

应用软件的具体情况​

技术指标​

集群工作节点支持1500以上的集群节点，支持24000张卡以上。

支持5000个用户同时在线使用使用云平台。支持工作节点的上线，下线替换等。

接口特性​

接口均为软件系统内部相互之间调用，前后端分离，不涉及外部接口访问。可进行网络隔离与分离。

内部接口采用HTTP RESTfull 形式，支持GET、POST等多种接口匹配。无其他诸如RPC之类的调用方式。

接口配备安全校验与鉴权检查。非鉴权用户不可进行资源与数据交互。

功能特性​

支持AI算力卡的指标检测，动态隔离故障节点，按需调度与资源分配，具备完整的云相关的功能。

部署方式​

离线部署

系统软件和购置的第三方软件的情况（含功能、性能等指标）​

系统软件：操作系统选用稳定性和性能优秀的Ubuntu Server LTS 2022.04.1版本，支持多种类型的CPU架构，并且长期支持版本更新以及bug修复，在部署OpenStack云、K8s集群以及50000个节点的大型渲染场等场景中，都能提供高性价比的横向扩展能力。功能等可以参考Ubuntu官方网站、Ubuntu官方文档。

第三方软件：本项目未采购第三方其他软件服务。
